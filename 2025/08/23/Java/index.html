方法和方法的重载

​ 方法就是一段用来完成特定功能的代码片段。（C语言中的函数）

方法的调用

方法的重载：

数组

​ 数组是相同类型数据的有序的集合。每一个数据称作一个元素，每个元素可以通过一个索引（下标）访问。

特点

  1. 长度确定，大小不可以被改变。
  2. 元素必须为相同类型。
  3. 数组类型可以为任意数据类型。

面向对象

类和对象的关系

类:相当于一个模板。对对象向上抽取出像的部分、公共部分形成类。

对象：模板下具体的产物。对象是一个个具体的实例。

​ 在Java中，先定义类，再创建对象。

类的编写、对象的创建及使用

类的编写：

  1. 取见名知意的名字，首字母大写，驼峰命名原则。
  2. 类的特性编写：特性就是类的属性部分。
  3. 类的行为编写：行为就是类的方法部分。

对象的创建和使用

  1. 格式：类名 对象名 = new 类名（）；
  2. 给属性赋值：对象名.属性名=值；
  3. 调用对象方法：[返回值类型 名字 = ]对象名.方法名（参数列表）

构造器

构造器作用：

​ 构造器是一个创建对象时被自动调用的特殊方法，用于对象初始化。构造器通过new关键字调用，从而返回该类的实例。

​ 构造器底层帮我们创建对象，在创建对象以后对对象进行初始化操作。如果类中没有显式的构造器，那么系统会默认分配一个空的构造器。调用构造器以后，对对象进行初始化，把对象的地址返回。

​ 尽量保证空构造器的存在，避免以后学到框架，没有添加空构造器时就会报错。如果一个类里有构造器，则不会被分配空构造器。

​ 构造器的参数名字如果和属性重名，则会发生就近原则。如果重名了，像给属性赋值，则需要在属性变量前加上this.进行修饰。

封装

封装思想：把该隐藏的隐藏，该暴露的暴露。追求高类聚、低耦合。隐藏对象内部的复杂性，只对外公开简单的接口。便于外界调用。

​ 高类聚：类的内部数据操作细节自己完成，不允许外部干涉。

​ 低耦合：仅对外暴露少量的方法用于使用。

继承

类是对对象的抽象、继承是对类的抽象。

首先定义一个Person的父类

子类通过关键词extends继承父类

子类可以进行属性和方法的扩展。

好处：

  1. 提高代码的复用性
  2. 便于代码的扩展
  3. 为了多态的使用，是多态使用的前提

方法的重写

​ 方法的重写建立在继承的条件下。当子类对父类方法不满意，则子类可对父类方法进行重写。方法的重写有严格格式要求，要求子类的方法名和父类一致，参数列表也要一致

重写和重载的区别

​ 重载：在同一个类中，方法名相同，参数列表不同，多个方法构成重载。

​ 重写：在不同类中，子类对父类方法不满意，对父类方法进行重写。

多态

​ 多态顾名思义就是多种形态，具体点就是去完成某个行为，当不同对象去完成时会产生不同状态。同一种行为，不同的子类呈现出状态不同。

​ 多态和属性无关，多态指的是方法的多态，不是属性的多态。

多态三要素：继承、重写、父类引用指向子类对象

多态好处：提高代码扩展性

异常

异常的处理

​ 异常就是在程序的运行过程中发生的不正常的事件，会中断正在运行的程序。Java中提供异常处理机制，将异常处理代码和业务代码分离，使程序更优雅、更好的容错性、高健壮性。

​ Java异常处理是通过五个关键字实现：try 、catch、 finally 、throw、 throws。

try-catch执行情况：当try中出现异常，catch中异常类型匹配，则执行catch代码块，并且执行catch之后的代码块，不执行try。当try有异常，但是catch中异常类型不匹配，则程序会中断。

throw throws:

​ 使用throw抛出异常：

两种解决方法：

  1. try –catch

2.添加异常到方法签名

throw与throws的区别：

  1. 位置不同：throw在方法内部，throws在方法的签名处。
  2. 内容不同：throw+异常对象，throws+异常类型。
  3. 作用不同：throw制造异常，throws告诉方法的调用者出现异常，调用者对异常进行处理。
