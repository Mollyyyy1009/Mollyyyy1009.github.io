区块链基本概念回顾

账户与钱包-私钥、公钥与地址的生产方式

核心概念与关系

从上至下，单项推导：私钥 → 公钥 → 地址

私钥：随机产生的一个字符串，包括字母和数字。。谁拥有私钥，谁就拥有账户的所有权。

公钥：根据私钥计算。由私钥通过椭圆曲线密码学 计算得出。可以公开，用于验证签名。 Public_Key = Private_Key * G

地址：由公钥通过哈希函数 (Keccak-256) 计算得出。可以公开，作为接收资金的标识。Keccak-256 哈希函数（以太坊），或者 RIPEMD-160 和 SHA-256（比特币）。

钱包

​ 钱包不是装币的容器，而是管理私钥的工具。

分类：

  1. 非托管钱包：你真正拥有私钥。私钥由你本地生成和存储（如MetaMask、Ledger、Trezor）。你的资产完全由你控制。
  2. 托管钱包：交易所（如Binance, Coinbase）为你持有私钥。你使用账户密码登录。你的资产由交易所托管，风险与此并存。

类型：纸钱包、冷钱包、热钱包、网页钱包、手机钱包、硬件钱包、交易所、随即钱包、种子钱包

私钥保管方式：助记词——钱包的主钥匙

​ 助记词是一组12或24个英文单词（来自BIP39标准单词列表）。通过一种确定性算法（BIP39），这一组助记词可以推导出无数个私钥和地址。这意味着你只需要备份这一组单词，就可以恢复你整个钱包（所有账户和资产）。

​ 助记词通过 PBKDF2 函数生成一个种子 (Seed)，这个种子再通过分层确定性 (HD) 结构生成所有私钥。所以：助记词 → 种子 → 主私钥 → 无数子私钥 → 无数公钥 → 无数地址

区块链浏览器

​ 区块链浏览器是一个可以查询区块链上所有公开数据的网站。区块链是公开透明的分布式账本。浏览器就是这个账本的交互式可视化界面。可以通过它查看任何一笔交易、任何一个地址的余额、任何一个区块的详情以及智能合约的活动。

Provider/Relay Network

  • Infura基本使用
  • 其他主网提供的节点

​ 简单来说，Provider 是一个允许你的应用程序（如钱包、DApp）与区块链网络通信的桥梁，是一个提供了 JSON-RPC 接口的服务器节点。应用程序通过向这个接口发送请求（如“获取这个地址的余额”、“发送这笔交易”）来与区块链交互。在 MetaMask 的网络设置中，填写的 “RPC URL” 就是 Provider 的地址。

​ 如果说 Provider 是查询的网关，那么 Relay Network 主要是为了解决“交易提交”的公平性和效率问题。中继网络是一个介于用户（发送者）和区块构建者（Builder）之间的网络。它的核心目的是接收交易，并隐私地、公平地将其传递给区块构建者，避免交易信息在进入公共内存池时被MEV机器人利用。

​ 所有的 Relay 都可以看作是一种特殊功能的 Provider，但并非所有的 Provider 都是 Relay。

NFT交易市场简介

​ NFT交易市场是一个专门用于买卖、交易NFT（非同质化代币）的线上平台。所有交易都以加密货币进行，并通过智能合约自动执行。

​ 它的核心功能是为NFT的创造者（Creator）和收藏者（Collector）提供一个相遇、展示和交易的场所。
