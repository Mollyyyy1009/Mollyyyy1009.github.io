2.值类型

Solidity中的变量类型

  1. 值类型：这类变量赋值时直接船体数值。
  2. 引用类型：占空间大，赋值时候直接传递地址（类似指针）。
  3. 映射类型：Solidity中存储键值对的数据结构，可以理解为哈希表。

值类型

1.布尔型

​ 二值变量，取值为true或false

​ 运算符：! 非 && 和 || 或 == 等于 != 不等于

注意： &&和||遵循短路规则。意思是，当出现&&时，第一个条件为false时就不会再去判断第二个条件，当出现*||时，第一个条件为true*时就不再去判断第二个条件。

2.整型

​ 整型是soliditty中的整数，常用的整型如下：

​ 其中，intN（有符号整数）的取值范围为：-2^(N-1)至2^(N-1) - 1。uintN（无符号整数）的取值范围为0至2^(N-1) - 1。

​ 指定数的大小可以节省Gas费用，降低交易成本。同时让语义明确。

常用的整型运算符包括：

  • 比较运算符（返回布尔值）： <=， <，==， !=， >=， >
  • 算术运算符： +， -， *， /， %（取余），**（幂）

3.地址类型

  • 普通地址：存储一个20字节的值（以太坊地址大小）
  • payable address：比普通地址多了transfer 和 send 两个成员方法，用于接收转账。

4.定长字节数组

  • 定长字节数组: 属于值类型，数组长度在声明之后不能改变。根据字节数组的长度分为 bytes1, bytes8, bytes32 等类型。定长字节数组最多存储 32 bytes 数据，即bytes32。
  • 不定长字节数组: 属于引用类型，数组长度在声明之后可以改变，包括 bytes 等。

3.函数

​ 函数形式：

  • function函数关键字，要声明函数则需要以此开头。
  • <function name>是函数名，跟在关键字function后。
  • ()括号内写入需要传入的参数，格式为：变量类型+名称。
  • 函数可见性说明符：
    • public：内外部可见。
    • private：只可以本合约内部访问，继承合约也无法使用此类函数。
    • external：只可以外部访问，内部可以通过this.function name()调用。
    • internal：只可以本合约内部访问，继承合约可以使用此类函数。
  • 决定函数权限功能关键字：
    • payable：可支付的。
    • pure：既不能读取也不能写入链上状态变量。
    • view：能读取但不能写入状态变量。
  • virtual用于父合约，标识的方法可以被子合约重写。override用在子合约上，表名方法重写了父合约的方法。
  • modifiers为自定义修饰器，数量不限。
  • return为函数返回的变量类型和名称。
  • {}中为函数体。

4.函数输出

  • return：跟在函数名后面，声明返回的变量类型及变量名。
  • returns：函数主体中返回指定的变量。

命名式返回

​ 在returns中标明返回变量的名称，solidity会初始化变量并且自动返回变量的值，不需要再使用return。

​ 声明返回变量类型与名字后，在函数内只需要给变量赋值即可。也可以用return进行返回。

解构式赋值

​ 解构式赋值规则用于读取函数的全部或者部分返回值。

  • 读取所有的返回值：声明变量，变量之间用,隔开，按顺序排列

  • 读取部分返回值：不读取的留空

5.变量

  • 状态变量
  • 局部变量
  • 全局变量

1.状态变量

​ 状态变量指其值被永久存储在智能合约的storage中的变量。一旦被写入，状态变量的值会永久保存在区块链上，直到被后续交易修改。

​ 在合约内部，所有函数都可以读取和修改它（除非有权限限制）。在合约外部，也可以通过调用函数来读取或间接修改（如果函数是 public 或 external）。

​ 写入状态变量操作需要支付Gas费。读取操作在合约内部是免费的，但从外部读取（通过调用view函数）通常也是免费。

2.局部变量

​ 局部变量是在函数或代码块（如 for 循环）内部声明的变量。它的生命周期和作用域被严格限制在声明它的函数或代码块内部。

​ 它们仅在函数执行期间存在。一旦函数调用结束，这些变量就会被销毁，它们的值不会保存在区块链上。

​ 创建和操作局部变量几乎不消耗 Gas（只有在某些复杂计算时会消耗极少量的计算Gas，但完全没有存储Gas成本）。

​ 访问局部变量只能在声明它们的函数或代码块内部被访问。其他函数甚至同一个合约中的其他函数都无法访问它们。

3.全局变量

​ 全局变量是一系列由以太坊虚拟机（EVM）和区块链本身提供的预定义、只读的特殊变量和函数。它们不是由开发者声明的，而是天然存在的。

​ 它们是语言和环境的一部分，无需也无法通过 uint、string 等关键字来声明。它们提供了访问区块链核心属性和交易关键信息的途径，例如：谁发送了交易、交易发送了多少钱、当前时间戳、当前区块的难度等。

​ 几乎所有全局变量都是只读的（constant），无法给它们赋值。它们的作用是提供信息，而不是存储信息。

​ 可以在合约的任何地方（状态变量初始化、函数、修饰器、构造函数等）直接访问全局变量。
